## Description

Service d'authentification utilisant NestJS, Better Auth et MongoDB.

## Fonctionnalités

- ✅ Authentification email/password avec Better Auth
- ✅ Gestion des sessions
- ✅ MongoDB comme base de données
- ✅ Validation des DTOs avec class-validator
- ✅ Configuration via variables d'environnement
- ✅ CORS activé
- ✅ Logs structurés

## Prérequis

- Node.js >= 18
- pnpm
- MongoDB >= 6.0 (local ou distant)

## Installation

```bash
# Installer les dépendances
$ pnpm install

# Copier le fichier .env.example vers .env
$ cp .env.example .env

# Modifier les variables d'environnement dans .env
# DATABASE_URL, BETTER_AUTH_SECRET, etc.
```

## Configuration

Créez un fichier `.env` à la racine du projet :

```env
DATABASE_URL="mongodb://127.0.0.1:27017/auth-service"
PORT=3000
NODE_ENV=development
BETTER_AUTH_SECRET="your-secret-key-change-in-production"
BETTER_AUTH_URL="http://localhost:3000"
CORS_ORIGIN="*"
```

## Démarrage de MongoDB

```bash
# Avec Docker
$ docker run -d -p 27017:27017 --name mongodb mongo:latest

# Ou utilisez votre installation locale de MongoDB
```

## Démarrage du projet

```bash
# Mode développement
$ pnpm run start:dev

# Mode production
$ pnpm run build
$ pnpm run start:prod
```

## API Endpoints

### Inscription
```bash
POST /auth/signup
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123",
  "name": "John Doe"
}
```

### Connexion
```bash
POST /auth/signin
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123"
}
```

### Déconnexion
```bash
POST /auth/signout
Authorization: Bearer <token>
```

### Récupérer la session
```bash
GET /auth/session
Authorization: Bearer <token>
```
